# TorchScript Custom Op Parser

A research-grade tool that walks the TorchScript IR of any PyTorch model — including models with user-defined operators — and emits a self-contained C++ inference header. The tool directly addresses the open gap in CERN's [SOFIE](https://root.cern/doc/master/group__TMVA__SOFIE.html) framework: automatic lowering of custom operators to standalone, dependency-free C++ inference code.

```
torch.nn.Module  ──jit.script──-  TorchScript IR  ──walk──-  IRGraph  ──emit──-  model_inference.h
                                                                  │
                                                        JSON IR serialisation
                                                      (torch-free round-trip)
```

---

## Motivation

SOFIE translates trained ML models into fast C++ functions for HEP inference workloads. Its documented limitation is **custom operator support**: operators defined outside the standard ONNX opset cannot be automatically lowered today. This tool solves that problem for the PyTorch/TorchScript ecosystem by detecting any non-`aten::` operator, extracting its full signature from the IR, and generating a typed C++ stub that is the exact extension point SOFIE requires.

---

## Features

- **Full IR walk** — inlines sub-graphs, walks all blocks, handles `prim::` and `aten::` node kinds
- **Custom op detection** — any non-`aten::` namespace is flagged; typed C++ stubs are auto-generated
- **Forward shape propagation** — concrete and symbolic dims tracked through the graph
- **JSON IR round-trip** — serialize the parsed graph and regenerate C++ without PyTorch installed
- **`--ops` flag** — register custom operators before parsing via an external `.ops.py` file
- **CMake generation** — `--cmake` emits a complete `CMakeLists.txt` targeting LibTorch
- **Extensible handler registry** — `@NodeHandler.register("ns::op")` to add handlers for new operators
- **Optional test harness** — `#ifdef TEST_MAIN` guarded `main()` with random-input smoke test

## CLI

```bash
# Parse a .pt file and emit a C++ header
python src/ts_parser.py parse \
    --model     resnet18.pt \
    --output    include/resnet18.h \
    --namespace cern::inference \
    --test-main \
    --cmake

# Parse a model with custom operators
python src/ts_parser.py parse \
    --model     custom_net.pt \
    --ops       custom_net.ops.py \
    --output    include/custom_net.h \
    --namespace cern::inference

# Regenerate C++ from a saved IR (no torch required)
python src/ts_parser.py codegen \
    --ir     model_ir.json \
    --output include/model.h

# Print a JSON summary of operators, shapes, and custom ops
python src/ts_parser.py inspect --model mymodel.pt

# Dump IR to JSON without emitting C++
python src/ts_parser.py dump --model mymodel.pt --ir model_ir.json
```

### The `--ops` file

For models with custom operators, provide a `.ops.py` file that registers them with `torch.library` before the `.pt` file is loaded:

```python
# custom_net.ops.py
import torch
import torch.library as lib

_lib = lib.Library("mylib", "DEF")
_lib.define("mish(Tensor x) -> Tensor")

@torch.library.impl(_lib, "mish", "CPU")
def mish_cpu(x):
    return x * torch.tanh(torch.nn.functional.softplus(x))

@torch.library.register_fake("mylib::mish")
def mish_fake(x):
    return torch.empty_like(x)
```

## Python API

```python
from ts_parser import parse_model, generate_header, save_ir, load_ir

graph = parse_model("mymodel.pt", ops_file="mymodel.ops.py")
save_ir(graph, "model_ir.json")

header = generate_header(
    graph,
    namespace         = "cern::inference",
    emit_test_main    = True,
    emit_custom_stubs = True,
)
open("include/model.h", "w").write(header)
```

## Example Output

Given a model containing `mylib::mish` and `aten::linear`, the tool produces:

```cpp
// AUTO-GENERATED by TorchScript Custom Op Parser
// Model  : custom_net
// Nodes  : 5  (1 custom)
// Custom ops:
//   mylib::mish
// DO NOT EDIT — regenerate with ts_parser.py
#pragma once
#ifndef INFERENCE_CUSTOM_NET_H
#define INFERENCE_CUSTOM_NET_H

#include <torch/torch.h>
#include <ATen/ATen.h>

namespace cern::inference {

// Custom operator stubs — fill in implementations below.

// Custom op stub: mylib::mish (appears 1x in graph)
inline at::Tensor mylib_mish(const at::Tensor& x0) {
    return x0;  // TODO: implement mylib::mish
}

static at::Tensor weight_1;  /* [512, 256], dtype=float */
static at::Tensor bias_1;    /* [512], dtype=float */

inline void load_weights(const std::string& weight_dir) {
    weight_1 = torch::load(weight_dir + "/weight_1.pt").to(torch::kFloat32).eval();
    bias_1   = torch::load(weight_dir + "/bias_1.pt").to(torch::kFloat32).eval();
}

// Forward pass
inline at::Tensor infer(const at::Tensor& x_1  /* [1, 256], dtype=float */) {
    auto _19 = torch::linear(x_1, weight_1, bias_1);
    auto _7  = mylib_mish(_19);  
    return _7;
}

}  
#endif  
```

Fill in the stub body and compile directly with LibTorch — no Python runtime, no ONNX export.

## Adding a Handler for a New Operator

```python
from ir_parser import NodeHandler, IRNode, IRValue, IRTensorShape
from typing import Dict, List, Optional

@NodeHandler.register("mylib::mish")
class MishHandler(NodeHandler):
    def infer_shapes(self, node: IRNode,
                     value_map: Dict[str, IRValue]) -> List[Optional[IRTensorShape]]:
        inp = value_map.get(node.inputs[0]) if node.inputs else None
        return [inp.shape if inp else None]  # element-wise: output shape = input shape

    def emit_cpp(self, node: IRNode, value_map: Dict[str, IRValue]) -> str:
        x, y = node.inputs[0], node.outputs[0]
        return f"auto {y} = {x} * torch::tanh(torch::nn::functional::softplus({x}));"
```

| `aten::matmul` | full | — |

## SOFIE Integration

The generated header slots directly into ROOT analysis code:

```cpp
#include "mymodel_inference.h"

void MyAnalysis() {
    cern::inference::load_weights("path/to/weights");
    auto input  = torch::from_blob(data_ptr, {1, 3, 224, 224});
    auto output = cern::inference::infer(input);
}
```

| Capability | SOFIE (ONNX path) | This tool |
|---|---|---|
| Standard ops | yes | yes |
| Custom op detection | no | yes |
| Custom op stubs | no | yes, auto-generated |
| Input format | ONNX | TorchScript `.pt` |
| Torch-free codegen | N/A | yes, via JSON IR |
| Shape annotations | partial | yes, per-line |
| Test harness | no | yes |
